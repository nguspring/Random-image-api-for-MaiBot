# Random Image API 插件

> 当前插件市场发布版本：**v1.0.0**

> 基于 [Random-image-api](https://github.com/inliver233/Random-image-api) 的 MaiBot 随机 Pixiv 图片插件

## 📖 简介

Random Image API 插件通过对接 Random-image-api 服务，为麦麦提供高质量的 Pixiv 随机图片获取功能。支持标签组合搜索、多图获取、AI 过滤、长宽比筛选等高级功能。

## ✨ 功能特性

- 🎨 **Pixiv 图源** — 所有图片来自 Pixiv
- 📦 **多图获取** — 一次最多获取 10 张图片
- 🔍 **强大的搜索功能**:
  - 标签包含 / 排除筛选
  - 指定画师 UID
  - 横屏 / 竖屏 / 方形筛选
  - 最低收藏数 / 浏览数 / 宽高 / 像素数筛选
- 🤖 **AI 作品控制** — 默认排除 AI 作品，可手动切换
- 🔞 **R18 内容控制** — 管理员总开关，默认关闭
- 📦 **两种发送模式** — 合并转发（整洁不刷屏）或逐条发送（直观展示）
- 🚀 **快速发图** — 绕过 VLM 识图，图片下载完直接发送，不等待模型处理
- 📐 **智能压缩** — 自动压缩大图再发送，2K 原图秒变几百 KB，发送速度提升数十倍
- 🛡️ **冷却管理** — 防止刷屏的冷却时间控制

## 📦 安装

1. 把插件文件夹放到 `MaiBot/plugins/` 目录下：

```
MaiBot/plugins/random_image_api_base64/
├── _manifest.json    # 插件元数据（不要动）
├── plugin.py         # 插件代码（不要动）
├── config.toml       # 配置文件（第一次启动后自动生成）
└── README.md         # 就是你在看的这个文件
```

2. 确保麦麦版本 ≥ `0.11.0`
3. 重启麦麦
4. 第一次启动后，系统会自动生成 `config.toml`
5. 打开 `config.toml`，把 `enabled` 改成 `true` 启用插件
6. 按需修改其他配置后再次重启麦麦

> ⚠️ **不需要手动安装额外的 Python 包**，插件声明了 `Pillow` 依赖，框架会在首次启动时自动安装。

> ⚠️ **插件默认是关闭的**，安装后需要手动在 `config.toml` 中把 `enabled` 改成 `true` 才能使用。

## ⚙️ 配置

编辑 `config.toml` 文件进行配置（第一次启动后自动生成）：

### 插件开关

```toml
[plugin]
# 插件总开关
# - true: 插件正常运行
# - false: 插件完全禁用，所有功能都不生效
# ⚠️ 默认是 false，需要手动改成 true 才能用！
enabled = false

# 配置版本号，不要手动修改
config_version = "1.0.0"

[components]
# 命令组件开关
# - true: 可以在聊天中使用"随机图片"等命令
# - false: 命令不可用（但插件本身还在运行）
enable_command = true
```

### API 配置

```toml
[api]
# Random-image-api 服务地址（末尾不要加 /）
# 默认使用公开地址，如果你自建了服务可以改成你的地址
base_url = "https://i.mukyu.ru"

# 请求超时时间（秒）
# 如果网络不好可以适当调大，建议 15-60
timeout = 30

# Cloudflare Worker 代理地址（加速图片下载）
# - 国内直连 i.mukyu.ru 可能很慢，CF Worker 可以加速
# - 默认由插件作者提供，可能随时失效，建议自建
# - 自建方法：https://dash.cloudflare.com → Workers → 创建 Worker
# - 留空则不使用代理，直接连接 API
cf_worker_url = "https://mukyu-proxy.1050786239.workers.dev"
```

### 功能配置

```toml
[features]
# 默认获取数量（1-10）
# 用户不指定数量时，默认获取几张图
default_num = 1

# 最大获取数量（1-10）
# 用户最多一次能要几张图，防止一次要太多
max_num = 10

# 冷却时间（秒），0 = 不限制
# 同一个用户两次使用命令之间的最短间隔
cooldown_seconds = 10

# R18 管理员总开关
# - false: 用户输入 r18 也不会启用（推荐）
# - true: 允许用户通过输入 r18 获取 R18 内容
allow_r18 = false

# 默认排除 AI 作品
# - true: 默认只返回人类画师作品（推荐）
# - false: 默认不过滤，AI 和人类作品都会出现
# 注意：无论此项怎么设置，用户都可以通过输入 "ai" 或 "noai" 临时覆盖
exclude_ai = true

# 默认搜索策略
# - "random": 纯随机（推荐，速度快）
# - "quality": 质量优先（会挑收藏高的图，但响应较慢）
default_strategy = "random"

# 发送模式
# - true: 合并转发格式（推荐，整洁不刷屏，像"查看聊天记录"那样）
# - false: 逐条发送（每张图单独一条消息，多图时会刷屏）
use_forward_message = true

# ========== 图片压缩配置 ==========

# 是否启用图片压缩
# - true: 下载图片后自动压缩再发送（推荐开启，大幅提升发送速度）
# - false: 发送原图，大图会很慢
compress_enabled = true

# 图片最大宽度（像素）
# - 超过这个宽度的图片会等比缩小
# - 在手机 QQ 上看，1280 已经非常清晰
# - 数字越小图片越小、发送越快，但清晰度会降低
# - 建议范围：800-2560
max_image_width = 1280

# JPEG 压缩质量（1-100）
# - 数字越大越清晰，但体积越大
# - 85 是公认的甜点值：肉眼看不出区别，体积压掉 80-90%
# - 建议范围：70-95
jpeg_quality = 85


# ========== 图片尺寸筛选 ==========

# 最小宽度（像素），0 表示不限制
min_width = 0

# 最小高度（像素），0 表示不限制
min_height = 0

# 最小总像素数，0 表示不限制
# 例：1920x1080 = 2073600 像素
min_pixels = 0
```

## 🎮 使用方法

### 基础命令

```
随机图片              # 获取 1 张随机图片（或 default_num 张）
来张图 3              # 获取 3 张图片
随机图片 help         # 显示帮助信息
```

以下触发词都可以用：`随机图片`、`来张图`、`来张涩图`、`随机涩图`、`来点图`、`setu`

### 标签搜索（用 # 号）

```
随机图片 #原神              # 包含"原神"标签
随机图片 #碧蓝航线 #巨乳    # 同时包含两个标签（AND）
随机图片 -#AI               # 排除"AI"标签
随机图片 #原神 -#AI         # 包含原神，排除 AI
```

### 数量 + 标签组合

```
来张图 3 #原神              # 3 张原神图
随机图片 5 #碧蓝航线 noai   # 5 张碧蓝航线非 AI 图
```

### AI 作品控制

```
随机图片 noai         # 仅非 AI 作品（如果配置了 exclude_ai = true，这是默认行为）
随机图片 ai           # 仅 AI 作品（临时覆盖默认设置）
```

> 💡 如果你在配置里设了 `exclude_ai = true`，那么不加任何参数时就已经是排除 AI 的了，不需要每次都写 `noai`。

### 发送模式切换

```
随机图片 3 forward    # 强制使用合并转发格式发送（无论配置如何）
```

> 💡 `forward` 参数可以临时覆盖配置中的 `use_forward_message` 设置。如果配置里已经是 `true`，不需要额外加这个参数。

### 长宽比筛选

```
随机图片 横屏           # 横图（宽 > 高）
随机图片 竖屏           # 竖图（高 > 宽）
随机图片 方形           # 方图（宽 ≈ 高）
```

### 画师筛选

```
随机图片 uid:12345678   # 获取指定画师的作品
```

### 收藏 / 浏览阈值

```
随机图片 收藏>1000      # 最低 1000 收藏
随机图片 浏览>5000      # 最低 5000 浏览
随机图片 宽度>1920      # 最小宽度 1920px
随机图片 高度>1080      # 最小高度 1080px
随机图片 像素>2073600   # 最小像素数（1920×1080）
```

```
# 普通图片触发词（不含 R18）
随机图片 / 来张图 / 来点图 / 来点图片 / 来张图片

# 涩图触发词（自动请求 R18，需配置 allow_r18 = true）
涩图 / 来张涩图 / 来点涩图 / 随机涩图 / setu / 来张setu / 来点setu
```

> 💡 说"涩图"类触发词时，插件会**自动**在请求里加上 `r18=1`，无需再手动加 `r18` 参数。
> 如果管理员未开启 `allow_r18 = true`，即使说涩图触发词也不会返回 R18 内容。

也可以在普通触发词后手动加 `r18` 参数临时控制：
```
随机图片 r18            # 普通触发词 + 手动 r18 参数
来张涩图            # 自动 R18，无需额外参数
来张涩图 3         # 3 张自动 R18 图片
```

### 策略覆盖

```
随机图片 quality        # 质量优先
随机图片 random         # 纯随机
```

### 组合使用

所有参数可以自由组合，顺序不限：

```
来张图 3 #原神 noai 横屏              # 3 张原神非 AI 横图
随机图片 5 #碧蓝航线 收藏>1500        # 5 张碧蓝航线高收藏图
随机图片 #白丝 -#AI uid:12345678      # 指定画师的白丝图，排除 AI 标签
来张涩图 #FGO quality             # 自动 R18，FGO 质量优先
来张涩图 3 收藏>2000 横屏        # 3 张高收藏 R18 横图
来张图 3 forward #原神                # 3 张原神图，合并转发格式
```

## ⚠️ 注意事项

### 首次使用

- 插件安装后**默认是关闭的**
- 第一次启动麦麦后会自动生成 `config.toml`
- 你需要手动把 `enabled = false` 改成 `enabled = true`
- 改完后重启麦麦才能生效

### R18 内容

- 默认**关闭** R18 功能
- 启用前请确保：
  - ✅ 符合当地法律法规
  - ✅ 群组成员均为成年人
  - ✅ 已告知群成员此功能
  - ✅ 理解使用风险

### AI 作品过滤

- 默认**排除** AI 作品（`exclude_ai = true`）
- 用户可以通过 `ai` 参数临时查看 AI 作品
- 用户可以通过 `noai` 参数临时强制排除 AI 作品
- 如果你想默认不过滤，把 `exclude_ai` 改成 `false`

### 使用限制

- 单次最多获取 10 张图片（可通过 `max_num` 调整）
- 有冷却时间限制（默认 10 秒，可通过 `cooldown_seconds` 调整）
- 请勿滥用，对 API 服务器友好一点

## 🔧 故障排除

### 无法获取图片

1. 检查网络连接是否正常
2. 查看 API 是否可访问：https://i.mukyu.ru/random?format=json
3. 检查日志文件中的错误信息
4. 确认 `config.toml` 中的 `base_url` 是否正确

### 图片发送失败

1. 可能是图片 URL 失效，重试即可
2. 检查网络连接是否正常
3. 尝试增大 `config.toml` 中的 `timeout` 值

### 搜索无结果

1. 尝试更换关键词或标签
2. 减少筛选条件（比如去掉收藏阈值）
3. 某些小众标签可能库中没有
4. 先试最简单的 `随机图片`，确认基本功能正常

### 我用的是自建服务

把 `api.base_url` 改成你的地址即可：

```toml
base_url = "https://your-domain.example.com"
```
确保你的服务端 `/random` 可正常返回 JSON。
